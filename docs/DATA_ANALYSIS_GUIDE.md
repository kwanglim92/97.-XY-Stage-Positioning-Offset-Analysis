## 1. 기초 통계 지표 해석

XY Stage Positioning 분석의 출발점은 **통계 지표가 의미하는 장비 상태를 정확히 파악**하는 것입니다. 각 지표는 서로 독립적으로 보지 않고, **조합해서 해석**할 때 가장 유효합니다.

| 지표 | 수식 | 엔지니어 해석 포인트 |
|------|------|---------------------|
| **Mean (평균)** | Σx / n | 스테이지의 **계통 오차(Systematic Error)** 지표. 0에서 벗어날수록 기계적 오프셋(Offset)이나 캘리브레이션 오류 의심. X Mean ≠ 0이면 X축 스테이지의 영점이 틀어진 것. |
| **StdDev (표준편차, σ)** | √(Σ(x-μ)² / n) | 측정 **재현성(Repeatability)**의 지표. 동일 조건에서 반복 측정했을 때 값이 흩어지는 정도. σ가 크면 스테이지 진동, 인코더 노이즈, 기계적 유격(Backlash) 등을 의심. |
| **Range (범위)** | Max − Min | 가장 극단적인 오차폭. 단 하나의 이상 측정값(Outlier)에도 민감하게 반응하므로 Outlier 제거 후에도 Range가 큰지 함께 확인해야 함. |
| **CV% (변동계수)** | (σ / \|μ\|) × 100 | Mean 대비 상대적 산포. Mean이 0에 가까울 때 CV%는 매우 민감하게 반응하므로, Mean이 극소값일 때는 이 지표 단독 해석에 주의. |
| **Out (이상치 수)** | IQR 기준 탐지 | Q1 − 1.5×IQR ~ Q3 + 1.5×IQR 밖의 데이터 수. Out > 0 이면 특정 Lot/시점에 국소적 이벤트(기계 충격, 진동, 척 오류 등)가 발생했음을 시사. |
| **Cpk (공정능력지수)** | min[(USL−μ)/3σ, (μ−LSL)/3σ] | 규격 중심을 고려한 공정 능력. **1.33 이상이 합격 기준**(4σ). 1.0~1.33은 경계, 1.0 미만은 불합격. Cpk가 낮은데 σ는 작다면, Mean이 규격 중심에서 치우쳤으므로 영점 재조정 필요. |

### 복합 해석 사례

| 상황 | 엔지니어 해석 및 조치 |
|------|--------------------|
| Mean 작음 + σ 큰 경우 | 평균 위치는 맞지만 **재현성이 나쁜 상태** → 스테이지 진동 또는 인코더 노이즈 조사 |
| σ 작음 + Mean 큰 경우 | 정밀하지만 **정확하지 않음(Accurate but not precise)** → 오프셋 보정(Calibration) 필요 |
| σ, Mean 둘 다 나쁜 경우 | 근본적인 **스테이지 정렬 실패** 검토 필요 |

## 2. Die별 평균 & Raw Deviation 분석

### 2.1 Die별 평균 히트맵 (Die Average Heatmap)

Wafer 전면을 격자로 분할한 **각 Die 위치에서의 평균 오차**입니다.

| 관찰 패턴 | 시각적 특징 | 가능한 원인 |
|-----------|-------------|-------------|
| **Edge Effect** | Wafer 가장자리 Die만 붉거나 파랗게 표시 | 척(Chuck) Edge 홀딩 불균일, 진공 흡착 불량, 가장자리 노광 편차 |
| **대각선 편차** | 특정 대각 방향으로 색상이 치우침 | 스테이지 Pitch/Yaw 틀어짐, 레일 마모 편차 |
| **중앙/주변 분리** | 중앙은 정상, 주변은 이상 (또는 반대) | 척 곡률(Bow/Warp) 문제, 웨이퍼 핀 기준점 오류 |
| **특정 Die 집중** | 단 1~2개 Die만 극단적 색상 | 해당 위치 인코더 이상, 해당 좌표 기구학적 이물질 |

> **StdDev 열과 함께 확인**: Avg만 크고 StdDev가 작으면 해당 위치의 오차는 **일관되고 반복적(구조적 문제)**. Avg와 StdDev가 모두 크면 **랜덤한 노이즈** 성분이 강한 것.

### 2.2 Raw Deviation 매트릭스

통계 처리 없이, **개별 측정(Repeat) × Die 위치**의 순수 측정값 행렬.

| 확인 방향 | 해석 내용 |
|-----------|-----------|
| **수평 방향 (행: Repeat)** | 특정 Repeat에서만 전 Die가 오차가 큰 경우 → 해당 측정 시점의 환경(진동, 온도 변화) 의심 |
| **수직 방향 (열: Die)** | Repeat이 반복될수록 오차가 커지는 경우 → 해당 위치의 **열팽창(Thermal Drift)** 또는 **기계적 유격 누적** 의심 |

## 3. XY Scatter (산점도) 심화 해석

XY Scatter는 **X 오차 vs Y 오차**를 2D 평면에 표현한 차트로, 스테이지의 **동작 특성을 가장 직관적으로 진단**할 수 있는 그래프입니다.

### 3.1 분포 형태별 진단표

| 분포 형태 | 시각적 설명 | 원인 및 조치 |
|----------|-------------|--------------|
| **Bull's-eye (동심원 집중)** | 점들이 원점 중심으로 조밀하게 모임 | ✅ 정상 상태. 스테이지 정밀도 우수 |
| **수직·수평 편위** | 점군이 X 또는 Y 한 방향으로만 치우침 | 해당 축 Linear Motor 또는 Encoder 오류, 오프셋 보정 필요 |
| **사선 형태 (Linear Trend)** | 45° 또는 임의 각도의 선형 분포 | X-Y 축 간 커플링(Coupling). 스테이지 **직각도(Squareness) 불량** 의심 |
| **다중 군집** | 2~3개의 점 그룹이 분리되어 존재 | Lot 간 재현 조건 차이(온도, 웨이퍼 로딩 변화) |
| **타원형 분포 (Ellipse)** | 타원형으로 넓게 퍼진 분포 | 두 축의 진동 크기가 다름. **기계 구동부의 강성(Stiffness) 불균형** 의심 |
| **Outlier 산재** | 대부분은 집중, 일부가 멀리 떨어짐 | 특정 Lot의 이상 이벤트. Raw Data의 Outlier 탭과 교차 확인 필요 |

### 3.2 Spec Guide Box 해석

그래프 중앙의 **빨간 점선 사각형(Guide Box)** = ± Spec Range 한계치

- 분포 중심(Centroid)이 박스 안에 있어도 **일부 점이 밖에 있으면** → 오차 Range 초과로 FAIL
- 박스가 매우 **좁고 점들이 빽빽하게 채운** 경우 → 장비가 Spec 한계선 경계에서 운용 중 → **Spec Margin 재검토 필요**
- **Log Scale 전환 시** → 극단적 Outlier의 영향을 줄이고 전체 분포 패턴 파악에 유리

## 4. Vector Map & Wafer Contour 분석

### 4.1 Vector Map (벡터맵) 패턴 진단

각 Die 위치에서의 **오차 벡터(크기 + 방향)**를 화살표로 표현. 단순한 크기뿐 아니라 **방향성 패턴**이 핵심.

| 벡터 패턴 | 엔지니어 해석 및 조치 |
|-----------|----------------------|
| **모든 화살표가 같은 방향** | 고정 오프셋(Constant Offset). Mean 값이 0에서 치우쳐있는 상태. **Calibration으로 즉시 수정 가능** |
| **방사형 (중앙→가장자리로 퍼짐)** | 스케일 오차(Scale Error). **EGA 스케일 파라미터 재보정** 필요 |
| **소용돌이형 (Swirl)** | 스테이지 회전(Rotation/Theta) 오차. **Theta 축 오프셋 조정** 필요 |
| **지역적 불규칙** | 특정 구역만 벡터 방향·크기 이상 → 해당 영역의 **척(Chuck) 핀 이상** 또는 **국소 기구 마모** 검토 |
| **랜덤 방향 (무작위)** | 구조적 패턴 없음 → 측정 재현성 자체가 부족. **진동 방지(Vibration Isolation)**나 측정 환경 개선 검토 |

> **Vector Scale 슬라이더 활용**: 실제 오차는 수 µm 이내라 육안 관찰이 어렵습니다. 슬라이더로 벡터 길이를 수십~수백 배 증폭하여 패턴을 명확하게 확인하세요. 단, **증폭된 길이 자체가 실제 오차 수치가 아님**에 유의하세요.

### 4.2 Wafer Contour (등고선 맵)

Wafer 전면의 **오차 분포를 연속적인 컬러 맵**으로 시각화. 측정하지 않은 Die 사이의 값은 보간(Interpolation)으로 채움.

| 색상 | 의미 |
|------|------|
| 🔴 붉은 계열 | 양(+) 방향의 큰 오차 구역 (High-Deviation Zone) |
| 🔵 청색/녹색 계열 | 음(-) 방향 또는 오차가 작은 구역 (Low-Deviation Zone) |

- **색상 전환이 급격한 구역** → **오차 구배(Gradient)가 급격**하게 변화. 해당 위치에 기구학적 변곡점 존재 가능성
- **X / Y 채널 교차 비교**: X는 정상이고 Y만 이상이라면 → Y축 전용 문제(인코더, 레일, 선형 모터) 격리 검토. 두 채널이 유사한 패턴이라면 → 공통 원인(기울어진 척, 하중 편심) 가능성.

## 5. Spec 판정 (PASS/FAIL) 기준 및 설정 로직

### 5.1 판정 흐름

| 단계 | 내용 |
|------|------|
| 1. 스캔 데이터 수집 | CSV 폴더에서 원시 수치 로드 |
| 2. Deviation Matrix 계산 | overall_range, overall_stddev 산출 |
| 3. Spec값과 비교 | overall_range ≤ spec_range **and** overall_stddev ≤ spec_stddev |
| 4. 최종 판정 | **두 조건 모두 충족** → ✅ PASS / **하나라도 초과** → ❌ FAIL |

### 5.2 Spec 설정 값 의미

| Spec 항목 | 의미 | 설정 가이드 |
|-----------|------|------------|
| **spec_range** | 허용되는 최대 오차 범위 (µm) | 고객/장비 사양의 Positioning Accuracy 값을 기준으로 설정 |
| **spec_stddev** | 허용되는 최대 표준편차 (µm) | Repeatability 사양 기준. 일반적으로 range의 1/3~1/5 수준 |

### 5.3 폴더명 불일치 에러 (⚠️ 가장 흔한 운영 오류)

측정 장비에서 내보낸 데이터 폴더 이름이 `settings.json`의 `standard_recipe_names`와 **정확히** 일치해야 합니다.

| 올바른 폴더명 ✅ | 잘못된 폴더명 ❌ |
|----------------|----------------|
| `Vision Pattern` | `1. Vision Pattern Recognize`, `VisionPttrn` |
| `In-Die Align` | `In Die Align`, `in_die_align` |
| `LLC Translation` | `LLC Trans`, `LLCtranslation` |
| `Global Align` | `4. Global Align`, `GlobalAlign` |

> 불일치 시 분석이 **원천 차단**됩니다. 탐색기에서 폴더 이름을 변경 후 재스캔 하세요.

### 5.4 Cpk와 Spec Range 연계 해석

Cpk는 `spec_limits`의 LSL/USL 기준이며, PASS/FAIL 판정은 `spec_deviation`의 `spec_range`/`spec_stddev` 기준입니다. 두 기준이 **독립적**으로 동작하므로, **Cpk가 높아도 Range가 Spec을 초과하면 FAIL**이 날 수 있습니다. 반드시 두 지표를 함께 확인하세요.

## 6. Lot Trend 차트 해석

Lot Trend는 시간 순서대로 Lot별 Mean 값 변화를 보여주는 차트입니다.

| 추세 패턴 | 의미 | 조치 |
|-----------|------|------|
| 📈 우상향 | Lot 진행 시 오프셋 증가 (드리프트) | 스테이지 캘리브레이션 점검 |
| 📉 우하향 | Lot 진행 시 오프셋 감소 | 자동 보정 동작 여부 확인 |
| ➡️ 수평 | 안정적 — 일관된 결과 | 정상 |
| 🔀 지그재그 | 불안정 — 랜덤 변동 | 외부 환경 요인 or 재현성 점검 |
| 단계적 변화 (Step) | 특정 Lot 이후 Mean 레벨이 급격히 변화 | 해당 시점의 장비 이벤트(유지보수, 부품 교체) 확인 |

> **Spec 라인 (± spec_range/2)**: 차트 상의 보라색 점선. Mean이 이 범위 안에 위지하도록 유지해야 합니다.
