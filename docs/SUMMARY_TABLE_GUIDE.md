# 📊 Summary 테이블 컬럼 가이드

> XY Stage Positioning Offset Analysis — 데이터 테이블 > Summary 탭

---

## 테이블 구조 개요

Summary 테이블은 각 Recipe(Step)별 측정 데이터의 **핵심 통계 요약**을 한눈에 보여줍니다.

```
┌────────┬───┬─────┬──────┬───────┬────────┬────────┬─────┬─────┬────┬────┬──────┬───────┐
│ Recipe │ R │  N  │ Mean │ Stdev │  Min   │  Max   │ CV% │ Out │ X  │ Y  │ 결과 │ Trend │
├────────┼───┼─────┼──────┼───────┼────────┼────────┼─────┼─────┼────┼────┼──────┼───────┤
│ Vision │2nd│ 440 │1104.2│1672.6 │-5051.3 │ 3249.5 │151.5│  0  │ ✅ │ ❌ │ FAIL │ 📈    │
│ In-Die │2nd│ 440 │1333.7│2974.3 │-2072.8 │ 4650.9 │223.0│  0  │ ✅ │ ✅ │ PASS │ 📈    │
│  ...   │   │     │      │       │        │        │     │     │    │    │      │       │
└────────┴───┴─────┴──────┴───────┴────────┴────────┴─────┴─────┴────┴────┴──────┴───────┘
```

## 컬럼별 상세 설명

### 1. Recipe

| 항목 | 내용 |
|------|------|
| **설명** | 측정 단계(Step)의 이름 |
| **값 예시** | `Vision Pattern`, `In-Die Align`, `LLC Translation`, `Global Align` |
| **결정 방식** | 폴더명에서 자동 추출 (예: `1. Vision Pattern` → `Vision Pattern`) |

> 폴더명이 `settings.json`의 `standard_recipe_names`와 일치하지 않을 경우 경고가 표시됩니다.

---

### 2. R (Round)

| 항목 | 내용 |
|------|------|
| **설명** | 측정 회차 (반복 측정) |
| **값 예시** | `1st`, `2nd` |
| **의미** | 동일 조건에서 재현성 확인을 위한 반복 회차 |

---

### 3. N (Data Count)

| 항목 | 내용 |
|------|------|
| **설명** | 해당 Recipe/Round에서 로드된 전체 데이터 개수 |
| **값 예시** | `440` |
| **산출** | X축 + Y축 데이터 합산 (Die 21개 × Lot 수 × 2축) |

---

### 4. Mean (평균값)

| 항목 | 내용 |
|------|------|
| **설명** | 전체 측정값(HZ1_O)의 산술 평균 |
| **단위** | nm (나노미터) |
| **의미** | 스테이지 포지셔닝 오프셋의 중심값 |
| **판단** | 0에 가까울수록 정밀도 높음. 큰 값은 계통 오차(Systematic Error) 가능성 |

---

### 5. Stdev (표준편차)

| 항목 | 내용 |
|------|------|
| **설명** | 측정값의 분산 정도 |
| **단위** | nm |
| **의미** | 값이 클수록 측정값 흩어짐이 큼 |
| **판단** | 작을수록 좋음. Spec 기준 `spec_stddev` 이하여야 PASS |

```
낮은 Stdev → 일관된 포지셔닝 (안정)
높은 Stdev → 불규칙한 포지셔닝 (불안정)
```

---

### 6. Min / Max (최솟값 / 최댓값)

| 항목 | 내용 |
|------|------|
| **설명** | 전체 측정값 중 최소/최대 |
| **단위** | nm |
| **의미** | 극단값 확인 → 이상치나 센서 오류 탐지 |
| **판단** | `Max - Min = Range`. Spec 기준 `spec_range` 이하여야 PASS |

---

### 7. CV% (변동계수)

| 항목 | 내용 |
|------|------|
| **설명** | Coefficient of Variation (변동계수) |
| **산출** | `(Stdev / Mean) × 100` |
| **의미** | 평균 대비 변동 비율 (단위 없는 상대적 비교 지표) |
| **판단** | CV% < 10%: 안정 / 10~30%: 보통 / > 30%: 불안정 |

> CV%가 높더라도 Mean이 0에 가까우면 수학적으로 커질 수 있으므로 다른 지표와 함께 판단.

---

### 8. Out (이상치 수)

| 항목 | 내용 |
|------|------|
| **설명** | IQR 방식으로 탐지된 이상치(Outlier) 개수 |
| **산출 방법** | 아래 참조 |
| **값 예시** | `0` = 이상치 없음, `3` = 3개 이상치 |

**IQR (Interquartile Range) 탐지 방식:**

```
1. 전체 값 정렬
2. Q1 (25%), Q3 (75%) 계산
3. IQR = Q3 - Q1
4. 하한 = Q1 - 1.5 × IQR
5. 상한 = Q3 + 1.5 × IQR
6. 하한 미만 or 상한 초과 → Outlier (is_outlier = True)
7. Out = True인 데이터 수
```

| Out 값 | 해석 |
|--------|------|
| **0** | 정상 — 모든 값이 IQR 범위 내 |
| **1~5** | 소수 이상치 — 특정 Die/Lot 확인 필요 |
| **>10** | 다수 이상치 — 측정 환경 점검 필요 |

---

### 9. X (X축 Pass/Fail)

| 항목 | 내용 |
|------|------|
| **설명** | X축 측정 결과의 Spec 적합 여부 |
| **표시** | ✅ = Pass, ❌ = Fail, — = 데이터 없음 |
| **판정 기준** | `X Range ≤ spec_range` **AND** `X StdDev ≤ spec_stddev` |

> Spec 기준값은 `settings.json` → `spec_deviation`에서 Recipe별로 설정.

---

### 10. Y (Y축 Pass/Fail)

| 항목 | 내용 |
|------|------|
| **설명** | Y축 측정 결과의 Spec 적합 여부 |
| **표시** | ✅ = Pass, ❌ = Fail, — = 데이터 없음 |
| **판정 기준** | X와 동일 (`Y Range ≤ spec_range` AND `Y StdDev ≤ spec_stddev`) |

---

### 11. 결과 (종합 판정)

| 항목 | 내용 |
|------|------|
| **설명** | X + Y 축 종합 Pass/Fail |
| **배경색** | 🟢 녹색 = PASS / 🔴 빨강 = FAIL / ⬜ 회색 = 데이터 없음 |

| X | Y | 결과 |
|---|---|------|
| ✅ | ✅ | **PASS** |
| ✅ | ❌ | **FAIL** |
| ❌ | ✅ | **FAIL** |
| ❌ | ❌ | **FAIL** |
| — | — | **—** |

---

### 12. Trend (Sparkline 미니 차트)

| 항목 | 내용 |
|------|------|
| **설명** | 해당 Recipe의 Lot별 Mean 값을 시간 순서대로 미니 라인 차트로 표시 |
| **크기** | 80 × 24 px |
| **렌더링** | QPainter 기반 실시간 렌더링 (SparklineTrendDelegate) |

**시각적 요소:**

| 요소 | 설명 |
|------|------|
| 파란 라인 | Lot 순서대로 Mean 값 변화 |
| 반투명 채우기 | 라인 아래 영역 (트렌드 방향 시각화) |
| 마지막 파란 점 | 가장 최근 Lot의 현재 상태 |

**패턴 해석:**

| 패턴 | 의미 | 조치 |
|------|------|------|
| 📈 우상향 | Lot 진행 시 오프셋 증가 (드리프트) | 스테이지 캘리브레이션 점검 |
| 📉 우하향 | Lot 진행 시 오프셋 감소 | 자동 보정 동작 여부 확인 |
| ➡️ 수평 | 안정적 — 일관된 결과 | 정상 |
| 🔀 지그재그 | 불안정 — 랜덤 변동 | 외부 환경 요인 or 재현성 점검 |

## Spec 설정 기준

X/Y Pass/Fail 판정에 사용되는 기준값은 `settings.json` → `spec_deviation`에서 관리합니다:

```json
"spec_deviation": {
  "Vision Pattern": { "spec_range": 1.0, "spec_stddev": 0.2 },
  "In-Die Align":   { "spec_range": 2.0, "spec_stddev": 0.4 },
  "LLC Translation": { "spec_range": 4.0, "spec_stddev": 0.8 },
  "Global Align":   { "spec_range": 6.0, "spec_stddev": 1.2 }
}
```

| 항목 | 의미 |
|------|------|
| `spec_range` | 허용 편차 범위 (µm) — `Max - Min` 기준 |
| `spec_stddev` | 허용 표준편차 (µm) |

> 두 조건 모두 충족해야 해당 축(X/Y)이 PASS 판정됩니다.
